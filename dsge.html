01 · Pendahuluan

Menghubungkan Teori dengan Data

Model DSGE (Dynamic Stochastic General Equilibrium) memiliki parameter-parameter struktural yang berasal dari perilaku optimisasi agen ekonomi — rumah tangga, perusahaan, dan pemerintah. Tantangan utamanya: bagaimana menentukan nilai parameter-parameter ini agar model mencerminkan realitas data?

Ada dua jalur utama yang digunakan peneliti: kalibrasi dan estimasi. Keduanya bukan pesaing, melainkan saling melengkapi tergantung konteks riset.

02 · Metode Pertama

Kalibrasi: Mencocokkan Steady State

Kalibrasi adalah pendekatan di mana peneliti memilih nilai parameter berdasarkan data jangka panjang rata-rata, studi mikro-ekonometrik, atau "fakta stilisasi" yang sudah mapan — bukan berdasarkan goodness-of-fit statistik terhadap data time series.

Definisi Formal
Kalibrasi adalah sekumpulan prosedur untuk memberikan jawaban atas pertanyaan ekonomi menggunakan model yang mengaproximasi Data Generating Process (DGP) dari data observasi. Parameter dipilih agar steady-state model mencerminkan rata-rata historis variabel makro.

Lima Langkah Kalibrasi (Kydland & Prescott, 1996)

1

Pose a Question

Rumuskan pertanyaan riset yang spesifik — apakah implikasi teoritis dari perubahan kebijakan, atau kemampuan model mereplikasi fitur data tertentu.

2

Gunakan Teori yang Telah Teruji

Pilih kerangka teoritis yang sudah terbukti memberikan jawaban reliabel untuk kelas pertanyaan serupa. Jangan membangun dari nol setiap kali.

3

Konstruksi Model Ekonomi

Pilih bentuk fungsional untuk primitif model (utility, produksi) dan cari solusi variabel endogen dalam bentuk variabel eksogen dan parameter.

4

Kalibrasi Model

Tentukan nilai parameter menggunakan target empiris pertama — misalnya rasio steady-state, elastisitas dari studi mikro, atau rata-rata historis. Ini bukan estimasi; nilai terpilih bukan yang memberikan fit terbaik secara statistik.

5

Jalankan Eksperimen

Evaluasi kualitas model dengan membandingkan outputnya terhadap set fakta stilisasi kedua. Lakukan analisis kebijakan jika diperlukan.

Contoh Kalibrasi: Model RBC Sederhana

Kunci Penting
Kalibrasi menggunakan satu set momen data untuk menentukan parameter, dan set kedua untuk menilai kinerja model. Diskrepansi yang ditemukan bisa menjadi uji atas teori yang digunakan — bukan kegagalan teknis.
03 · Metode Kedua

Estimasi: Menggunakan Seluruh Informasi Data

Estimasi menggunakan teknik ekonometrika formal untuk menentukan parameter — bukan hanya mencocokkan beberapa momen steady state, melainkan memanfaatkan seluruh informasi dalam data time series.

Masalah 1: Stochastic Singularity

Ketika model DSGE hanya memiliki satu sumber shock (misalnya shock teknologi), semua variabel observasi menjadi proporsional satu sama lain. Ini disebut stochastic singularity — model memprediksi bahwa rasio tertentu antara variabel akan selalu konstan, padahal data menunjukkan sebaliknya.

Syarat Identifikasi
Agar model memiliki estimasi ekonometrika yang well-defined: setiap variabel observable harus memiliki setidaknya satu shock unobservable. Ini bisa berupa measurement error atau shock struktural pada setiap persamaan.

Model State-Space: Jembatan ke Estimasi

Model DSGE log-linearisasi dengan campuran variabel observable dan unobservable adalah contoh state-space model, yang terdiri dari dua persamaan:

Transition Equation (Persamaan State)
St = F·St-1 + ut
St: variabel state unobservable; ut ~ N(0, Σu)
Measurement Equation (Persamaan Pengukuran)
Zt = H·St + vt
Zt: variabel observable; vt ~ N(0, Σv)

Maximum Likelihood Estimation (MLE)

Ketika semua variabel terobservasi, MLE mencari parameter yang memaksimumkan fungsi log-likelihood gabungan:

Combined Log-Likelihood
max log L = log LX + log LZ
Subject to: cross-equation restrictions dari struktur model

Untuk variabel unobservable, Kalman Filter digunakan untuk menghitung likelihood secara rekursif — Dynare dapat melakukan ini secara otomatis setelah model dispecifikasi.

Keterbatasan MLE
Menurut Fernandez-Villaverde: likelihood DSGE penuh dengan local maxima dan permukaan hampir datar. Data kuartalan yang terbatas memperparah masalah ini. Standard errors notoriusly sulit dihitung, dan distribusi asimtotik menjadi aproksimasi buruk untuk sampel kecil.
03b · Limited Information Estimation

IRF Matching: Estimasi Berbasis Impulse Responses

Selain kalibrasi penuh dan full-information Bayesian, ada pendekatan ketiga yang sangat berpengaruh dalam literatur DSGE modern: limited information estimation via IRF matching. Pendekatan ini digunakan oleh Christiano, Eichenbaum & Evans (2005) — model yang menjadi fondasi sebagian besar DSGE pre-krisis — dan dipertahankan karena fleksibilitas dan transparansinya.

Prinsip Dasar IRF Matching
Alih-alih memaksimumkan full likelihood atau mencocokkan seluruh momen data, IRF matching meminimumkan jarak antara IRF model dan IRF data — biasanya IRF yang diestimasi dari VAR — untuk subset shock tertentu (misalnya shock kebijakan moneter). Ini disebut "limited information" karena tidak mengeksploitasi seluruh implikasi model terhadap momen data.
Objective Function IRF Matching
minθ [IRFmodel(θ) − IRFVAR]' · W · [IRFmodel(θ) − IRFVAR]
W: matriks pembobot; IRFVAR dianggap sebagai "data" yang harus dicocokkan model

Mengapa Menggunakan VAR sebagai Benchmark?

IRF dari Vector Autoregression (VAR) digunakan sebagai target karena beberapa alasan. Pertama, VAR adalah representasi fleksibel dari data tanpa asumsi teori yang kuat. Kedua, metodologi identifikasi shock telah mapan — misalnya shock kebijakan moneter diidentifikasi dengan asumsi tidak ada dampak kontemporer pada output dan inflasi (Christiano, Eichenbaum & Evans, 1999). Ketiga, IRF VAR terbukti robust terhadap berbagai spesifikasi lag dan periode sampel.

Robustness IRF VAR (CET 2018)
Christiano, Trabandt & Walentin (2010) melakukan 700 estimasi VAR dengan kombinasi berbagai lag (n = 1,...,5) dan titik awal sampel (1951:Q1–1985:Q4). Temuan kunci: properti kualitatif IRF sangat robust — hump-shaped response GDP, konsumsi, dan investasi; kenaikan kecil inflasi; real wage yang hampir tidak terpengaruh. Ini menjadi bukti bahwa IRF VAR adalah target yang valid untuk evaluasi model DSGE.

Prosedur Estimasi: Dari GMM ke Bayesian Limited Information

1

Estimasi IRF dari Data (VAR)

Gunakan VAR dengan identifying assumptions yang tepat untuk mendapatkan IRFVAR beserta confidence intervalnya. Ini menjadi "data target" untuk tahap berikutnya.

2

Pilih Parameter yang Dikalibrasi vs Diestimasi

Parameter steady-state (β, α, δ) dikalibrasi dari data rata-rata jangka panjang. Parameter dinamika (Calvo probability, habit coefficient, investment adjustment cost) diestimasi dari IRF matching.

3

Minimisasi Jarak IRF (GMM atau Bayesian)

Minimumkan jarak antara IRFmodel(θ) dan IRFVAR. Pendekatan klasik menggunakan GMM (Hansen, 1982); pendekatan modern menggunakan Bayesian limited information (Christiano, Trabandt & Walentin, 2010) yang memungkinkan penerapan prior yang transparan.

4

Evaluasi: Apakah Model Bisa Mereplikasi IRF?

Plot IRF model vs IRF VAR dalam satu grafik. Jika IRF model berada dalam confidence band VAR, model dianggap valid untuk set momen tersebut. Laporkan parameter estimasi: mode posterior, mean, dan HPD interval.

Contoh Hasil: Model CEE (2005) via CET (2016)

Berikut parameter kunci yang diestimasi menggunakan IRF matching dalam model Christiano, Eichenbaum & Trabandt (2016) — re-estimasi dari CEE (2005):

ParameterInterpretasiPosterior ModeCatatan

ξw Calvo wage stickiness Ganti upah rata-rata sekali setahun Lebih sticky dari harga — krusial untuk akuralitas IRF

Peran Kritis Wage Stickiness
CET (2018) menekankan: ketika nominal wages diasumsikan fleksibel (holding semua parameter lain tetap), performa model memburuk drastis. Sticky wages adalah mekanisme yang membuat marginal cost bersifat acyclical, yang pada gilirannya menjelaskan mengapa inflasi hampir tidak bergerak meski ada shock moneter yang besar. Ini menunjukkan bahwa evaluasi sensitivitas terhadap asumsi kekakuan nominal adalah prosedur standar dalam evaluasi model DSGE.

Keunggulan dan Keterbatasan IRF Matching

Keunggulan
  • Transparan: target yang dicocokkan jelas terlihat
  • Fleksibel: bisa fokus pada satu jenis shock saja
  • Prior VAR bisa diverifikasi secara independen dari model
  • Menghindari stochastic singularity
  • Tidak memerlukan asumsi distribusi error eksplisit
Keterbatasan
  • Hanya menggunakan subset informasi dalam data
  • Pilihan momen target berpotensi arbitrary
  • Bergantung pada validitas identifikasi VAR
  • Model bisa fit baik untuk shock yang ditarget tapi buruk di lain tempat
  • Perlu konfirmasi dengan full-information evaluation
04 · Pendekatan Modern

Estimasi Bayesian: Standar Kontemporer

Karena keterbatasan MLE, pendekatan Bayesian telah menjadi standar dalam estimasi DSGE modern. Bayesian menggabungkan informasi dari data (likelihood) dengan pengetahuan prior (prior distribution) untuk menghasilkan posterior distribution parameter.

Teorema Bayes untuk DSGE
p(θ | Y) ∝ p(Y | θ) · p(θ)
Posterior ∝ Likelihood × Prior
Intuisi Bayesian
Dalam Bayesian: parameter adalah variabel random, data adalah tetap. Kita ingin tahu seberapa plausible suatu model relatif terhadap alternatifnya, given data yang kita amati. Ini berbanding terbalik dengan MLE di mana parameter adalah tetap dan data adalah random.

Pemilihan Distribusi Prior

Pemilihan prior mengikuti batasan teoritis parameter:

Normal
(-∞, +∞)
Parameter tak terbatas: koefisien respon kebijakan, elastisitas yang bisa negatif/positif
Gamma
(0, +∞)
Parameter yang harus positif: standar deviasi shock, parameter price markup, Frisch elasticity
Beta
(0, 1)
Parameter antara nol dan satu: probabilitas Calvo, discount factor β, koefisien autoregresif ρ

Proses Estimasi Bayesian di Dynare

1

Spesifikasi Prior

Peneliti menentukan distribusi prior untuk setiap parameter yang akan diestimasi, berdasarkan studi terdahulu dan pertimbangan teoritis.

2

Kalman Filter untuk Likelihood

Dynare mengubah model ke state-space form, lalu menggunakan Kalman filter untuk menghitung likelihood data untuk setiap set parameter yang dicoba.

3

Posterior Sampling (MCMC)

Metropolis-Hastings atau algoritma MCMC lain digunakan untuk mencuplik dari posterior distribution — mengintegrasikan likelihood × prior secara numerik.

4

Pelaporan Hasil

Hasil dilaporkan sebagai perbandingan posterior mean vs prior mean, disertai credible intervals (HPD intervals) dari distribusi posterior.

Keunggulan Bayesian vs MLE
Mengintegrasikan likelihood lebih mudah secara komputasional daripada memaksimumkannya (terutama untuk fungsi berdimensi tinggi). Bayesian juga lebih robust terhadap local maxima karena menggunakan keseluruhan distribusi, bukan satu titik.
06 · Peringatan Kritis

Masalah Identifikasi Parameter

Bahkan ketika estimasi Bayesian digunakan dengan benar, tidak semua parameter benar-benar "diestimasi dari data." Beltran & Draper (2008) dalam paper Fed Reserve mereka menunjukkan bahwa banyak parameter kunci DSGE bersifat weakly identified — artinya data tidak informatif dan posterior didominasi oleh prior.

Peringatan dari Beltran & Draper (2008)
Ketika prior dan posterior mendekati identik, klaim bahwa parameter "telah diestimasi" adalah menyesatkan. Menggunakan prior informatif dari literatur dalam kondisi ini sebetulnya hanyalah "cara rumit untuk mengkalibrasi" — data hampir tidak berkata apa-apa, dan hasil akhir terutama didorong oleh prior yang diasumsikan.

Parameter yang Cenderung Weakly Identified

Dari estimasi model small open economy Swiss (1970–2005), parameter-parameter berikut terbukti sulit diidentifikasi oleh data:

Alat Diagnostik Weak Identification

Beltran & Draper merekomendasikan serangkaian alat untuk mendiagnosis masalah identifikasi sebelum menginterpretasikan hasil estimasi:

1

Surface Plot Log-Likelihood

Plot log-likelihood sebagai fungsi dari dua parameter sekaligus. Parameter yang well-identified menghasilkan puncak yang tajam dan terdefinisi jelas; yang weakly identified menunjukkan permukaan hampir datar atau ridge line.

2

Heat Plot (3 Parameter)

Visualisasikan log-likelihood sebagai fungsi tiga parameter. Untuk parameter yang well-identified, hanya kombinasi parameter yang sempit yang menghasilkan nilai likelihood dalam 1% tertinggi.

3

Monte Carlo dengan Data Artifisial

Generate data dari model dengan parameter true θDGP yang diketahui, lalu estimasi ulang. Jika θ̂ ≈ θDGP, parameter teridentifikasi. Penting: butuh ~1000 observasi vs 142 data kuartalan nyata.

4

Estimasi dengan Tiga Set Prior

Estimasi dengan prior uniform, somewhat informative, dan informatif. Jika posterior berubah drastis antar-set, parameter weakly identified — prior mendominasi, bukan data.

Implikasi untuk Thesis DSGE
Untuk model dengan perfect foresight solver (tanpa Bayesian estimation), masalah identifikasi ini tidak langsung relevan — tapi justru ini yang memvalidasi pendekatan kalibrasi penuh. Parameter seperti β dan φ yang sulit diidentifikasi dari data siklus bisnis memang lebih tepat dikalibrasi dari studi mikro atau nilai konvensional literatur.
06b · Evaluasi Substantif

Kekakuan Nominal & Evaluasi Kuantitatif Model

Paper Christiano, Eichenbaum & Trabandt (2018) memberikan panduan substantif tentang bagaimana mengevaluasi apakah mekanisme model — khususnya kekakuan nominal — benar-benar penting secara kuantitatif. Ini relevan tidak hanya untuk model moneter, tetapi juga untuk model DSGE dengan frictions lain seperti collateral constraints atau financial accelerator.

Tiga Pilar Kekakuan Nominal dalam Model DSGE Modern

1

Calvo Price Stickiness (Nominal Price Rigidity)

Dalam model Calvo (1983), perusahaan hanya bisa mengubah harga dengan probabilitas eksogen (1 − ξp) setiap periode. Ini menangkap fitur data bahwa perusahaan merespons variasi permintaan lebih banyak dengan perubahan kuantitas daripada harga dalam lingkungan inflasi moderat. Calvo stickiness menciptakan hubungan antara inflasi dan ekspektasi biaya marginal masa depan (New Keynesian Phillips Curve).

2

Calvo Wage Stickiness (Nominal Wage Rigidity)

Serupa dengan harga, household/workers hanya bisa merenegosiasi upah dengan probabilitas (1 − ξw) setiap periode. Kekakuan upah ini secara mekanis memastikan biaya marginal perusahaan bersifat acyclical — tidak banyak berfluktuasi dengan siklus bisnis. Tanpa wage stickiness, model sulit mereplikasi fakta bahwa inflasi hampir tidak bergerak meski terjadi fluktuasi output yang besar.

3

Habit Formation & Investment Adjustment Costs

Kekakuan riil (bukan nominal) ini bekerja bersama kekakuan nominal untuk menghasilkan hump-shaped dynamics. Habit formation: utilitas marginal konsumsi bergantung positif pada level konsumsi masa lalu, mendorong household menaikkan konsumsi secara bertahap. Investment adjustment costs: mempermahal perubahan tingkat investasi, menghasilkan hump-shape investasi setelah shock.

Anti-Fisherian Property
Model New Keynesian standar memenuhi sifat anti-Fisherian: setelah shock moneter transitori ekspansif (penurunan suku bunga nominal), inflasi dan suku bunga bergerak berlawanan arah. Hal ini terjadi karena perusahaan menetapkan harga berdasarkan biaya marginal saat ini dan mendatang — shock transitori hampir tidak mempengaruhi kondisi masa depan, sehingga inflasi naik sedikit, sementara suku bunga turun. Akibatnya, suku bunga riil turun, mendorong konsumsi dan investasi naik. Sifat ini menjadi salah satu indirect test untuk memvalidasi mekanisme model.

Strategi Evaluasi Kuantitatif: Counterfactual Analysis

CET (2018) menunjukkan metode evaluasi yang ampuh: ubah satu asumsi, tahan semua parameter lain tetap pada nilai estimasi. Ini memungkinkan identifikasi kontribusi kausal setiap mekanisme terhadap kinerja model.

Eksperimen CounterfactualPerubahan AsumsiTemuan KunciImplikasi
Flexible wages ξw → 0 (upah fleksibel penuh) Performa model memburuk drastis; IRF tidak lagi cocok data Wage stickiness adalah mekanisme esensial, bukan ornamen
Flexible prices ξp → 0 (harga fleksibel penuh) Inflasi terlalu responsif terhadap shock Price stickiness diperlukan untuk mendampening inflasi
No habit formation b = 0 (time-separable utility) Konsumsi dan investasi tidak lagi hump-shaped Habit formation kritis untuk replicating dynamics
No investment adj. costs S'' = 0 (investasi tanpa biaya adjustment) Investasi melompat secara diskontinu setelah shock IAC diperlukan untuk hump-shaped investasi

Pelajaran untuk Evaluasi Model DSGE Lain

Metodologi counterfactual ini berlaku general — tidak terbatas pada model moneter. Dalam model bencana alam atau model dengan financial frictions, prosedur serupa dapat dilakukan:

Prinsip Evaluasi Model yang Baik (dari CET 2018)
  1. Tetapkan target IRF yang jelas: Apa "fakta stilisasi" yang harus direplikasi model? Tentukan sebelum estimasi.
  2. Laporkan posterior mode dan interval: Bukan hanya titik estimasi — credible intervals memperlihatkan uncertainty parametrik.
  3. Bandingkan prior vs posterior: Jika posterior ≈ prior, parameter tidak teridentifikasi dari data. Nyatakan secara eksplisit.
  4. Lakukan counterfactual analysis: Ubah satu mekanisme ke "off" dan lihat dampak terhadap IRF. Ini membuktikan atau menyangkal pentingnya mekanisme tersebut.
  5. Evaluasi out-of-sample: Model yang fit in-sample belum tentu valid. Uji forecasting atau evaluasi di periode berbeda.

Dari Frictions Finansial ke Frictions Bencana: Analogi Struktural

CET (2018) juga menunjukkan evolusi model DSGE pasca-krisis: penambahan financial frictions (BGG financial accelerator, collateral constraints à la Kiyotaki-Moore, shadow banking) yang sebelumnya dianggap perlu namun ternyata memiliki efek kuantitatif terbatas. Analoginya relevan untuk model dengan disaster shocks:

Financial Frictions (Post-GFC)
  • BGG financial accelerator: amplifikasi modest terhadap IRF standar
  • Kiyotaki-Moore collateral constraint: efek dinamis sangat kecil (Kocherlakota, 2000)
  • Pelajaran: inclusion tidak otomatis berarti quantitatively important
  • Diperlukan: financial data (spread, equity) sebagai observasi tambahan
Disaster/Spatial Frictions (Model Bencana)
  • Kolateral constraint (damaged capital): mekanisme amplifikasi analog BGG
  • Public capital destruction: persistent supply shock, analog productivity shock
  • Pelajaran yang sama: harus diuji apakah mechanism ini quantitatively material
  • Diperlukan: regional output, investment, government spending sebagai observasi target

Tiga Metode Aproksimasi Utama

Log-Linearisasi (Order 1)
  • Paling umum digunakan dalam literatur
  • Aproksimasi Taylor order pertama di sekitar steady state
  • Certainty equivalence berlaku — variansi shock tidak mempengaruhi nilai ekspektasi
  • Cocok untuk analisis IRF dan momen kedua
  • Digunakan Dynare secara default
Perturbation Order 2+
  • Diperlukan untuk analisis welfare yang akurat
  • Mengikutsertakan efek variansi shock (risk premium)
  • Certainty equivalence tidak berlaku
  • Lebih akurat jauh dari steady state
  • Dynare mendukung hingga order ke-6
Perfect Foresight Solver
  • Solusi deterministik untuk path variabel
  • Tidak memerlukan linearisasi
  • Ideal untuk analisis transisi dan kebijakan satu-kali
  • Tidak perlu appendix linearisasi dalam thesis
  • Dynare: simul atau perfect_foresight_solver

Metode Blanchard-Kahn: Syarat Eksistensi Solusi

Metode eigenvalue-eigenvector dari Blanchard & Kahn (1980) adalah standar untuk menemukan solusi rational expectations. Kondisi kunci untuk eksistensi dan keunikan solusi:

Blanchard-Kahn Condition
Solusi model rational expectations ada dan unik jika dan hanya jika jumlah eigenvalue di luar unit circle sama persis dengan jumlah variabel non-predetermined (forward-looking). Ini dikenal sebagai saddle path stability.
# unstable eigenvalues = # non-predetermined variables
Jika terlalu banyak: tidak ada solusi (explosive). Jika terlalu sedikit: solusi multipel (indeterminacy)

Mengapa Certainty Equivalence Penting untuk Kalibrasi

Dari Flotho (2009): pada aproksimasi order pertama, nilai ekspektasi variabel endogen sama dengan nilai steady state — variansi shock tidak mempengaruhi. Ini berarti kalibrasi parameter dari rata-rata data (steady state targets) sepenuhnya konsisten dengan solusi order pertama. Ini justifikasi teoritis mengapa kalibrasi dari data jangka panjang valid untuk model yang diselesaikan dengan linearisasi.

Certainty Equivalence (Order 1)

Ukuran variansi shock (σ) tidak mempengaruhi policy function pada aproksimasi order pertama

Evaluasi Model: Dua Cara Utama

Setelah kalibrasi/estimasi selesai, model dievaluasi dengan dua pendekatan yang saling melengkapi:

Matching Moments
  • Bandingkan standar deviasi dan korelasi variabel model vs data aktual
  • Hitung unconditional moments dari simulasi
  • Model bagus jika statistik mendekati data ekonomi nyata
  • Variance decomposition untuk relatif kepentingan shock
Impulse Response Functions
  • Mulai dari steady state, berikan satu shock ke variabel eksogen
  • Hitung time-path respons variabel endogen
  • Model bagus jika IRF menyerupai respons ekonomi nyata
  • Bisa dibandingkan dengan IRF dari SVAR sebagai benchmark

Kapan Menggunakan Apa?

Kalibrasi
  • Parameter menggunakan rata-rata historis jangka panjang
  • Studi mikro-ekonometrik (labor, preferensi)
  • Model sederhana / model dasar
  • Tidak perlu data time series panjang
  • Kontrol penuh atas "interpretasi" parameter
  • Umum: depreciation rate, discount factor, capital share
Estimasi (Bayesian)
  • Menggunakan seluruh informasi dalam data
  • Parameter seperti price stickiness, habit formation
  • Model medium-to-large scale (misalnya DSGE BI)
  • Memerlukan data time series yang cukup
  • Menghasilkan uncertainty bounds yang formal
  • Umum: Calvo parameter, Taylor rule coefficients
Praktik Terbaik dalam Literatur
Pendekatan modern umumnya menggabungkan keduanya: parameter yang menentukan karakteristik jangka panjang (steady state) dikalibrasi, sementara parameter yang menentukan dinamika siklus bisnis (shock persistence, nominal rigidities) diestimasi secara Bayesian. Namun perlu dicek identifikasi — jika data tidak informatif untuk suatu parameter, estimasi Bayesian hanyalah kalibrasi yang lebih rumit (Beltran & Draper, 2008). Dynare memfasilitasi keduanya dalam satu file .mod.
